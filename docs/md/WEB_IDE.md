# Nex Web IDE

This guide explains how to start and use the browser-based Nex IDE.

## What You Get

The Web IDE has 4 sections:

1. REPL: evaluate Nex expressions and statements interactively.
2. Nex Editor: write larger programs and run them in one click.
3. Canvas: render `Window` and `Turtle` graphics output.
4. Documentation: in-app tutorial pages.

## Prerequisites

- Java 11+
- Clojure CLI
- Node.js 16+ (for `shadow-cljs`)

## Start the Web IDE

From the project root:

```bash
./scripts/build-parser-runtime.sh
npx shadow-cljs compile browser
```

Serve the `public/` folder with any static file server, for example:

```bash
npx serve public
```

Then open the URL printed by `serve` (typically `http://localhost:3000`).

## Development Workflow

For live-reload style development, run:

```bash
./scripts/build-parser-runtime.sh
npx shadow-cljs watch browser
```

In another terminal, serve static files:

```bash
npx serve public
```

## Using the IDE

### 1) REPL

Enter any Nex input in the REPL input box and click `Evaluate`.

Examples:

```nex
1 + 2
```

```nex
let x: Integer := 100
print(x + 20)
```

### 2) Editor

Type a multi-line program in the editor and click `Run In REPL`.

Example:

```nex
let win := create Window.with_title("Nex Browser", 640, 360)
win.show()
let t := create Turtle.on_window(win)
t.color("blue")
t.forward(80.0)
```

### 3) Canvas

Graphics output appears in the Canvas panel automatically when `Window`/`Turtle` APIs are used.

### 4) Documentation

Use `Previous` / `Next` to browse built-in tutorial pages.

## Troubleshooting

### IDE shows old behavior

Rebuild and hard refresh:

```bash
./scripts/build-parser-runtime.sh
npx shadow-cljs compile browser
```

Then browser hard refresh (`Ctrl+Shift+R` / `Cmd+Shift+R`).

### Which static server command?

Use:

```bash
npx serve public
```

(`npx server public` is a different package and may behave differently depending on your environment.)

### Parser module errors in browser

If you see parser/module-missing errors in REPL:

1. Rebuild parser runtime with `./scripts/build-parser-runtime.sh`
2. Rebuild browser bundle with `npx shadow-cljs compile browser`
3. Hard refresh the page
4. Confirm you are loading the latest build marker shown in the REPL output

## Build Artifacts to Deploy

Deploy the full `public/` directory, including:

- `public/index.html`
- `public/js/` (shadow-cljs output)
- `public/parser-runtime/` (generated by `scripts/build-parser-runtime.sh`)

If you only deploy `public/js`, parser loading will fail in the browser.
